# Изменения по сравнению с оригинальным TAPChecker

Документ фиксирует отличия нового проекта «Ассистент преподавателя» от идеи исходного проекта [TAPChecker](https://github.com/GSatosov/TAPChecker) для корректного отражения в тексте дипломной работы.

---

## 1. Локальное создание заданий вместо Google Drive

**Изменение:** задания создаются и редактируются внутри приложения и сохраняются в локальный файл `data/assignments.json`. Тестовые данные не загружаются из Google Drive.

**Причина:** для защиты ВКР требуется автономное приложение, которое стабильно работает без интернета, OAuth-токенов и внешних облачных сервисов.

**Что изменить в дипломе:** в разделе проектирования (например, §2.4 «Хранение данных») указать, что в демонстрационной версии источником заданий является локальное хранилище, а интеграция с облаком вынесена в перспективу развития.

**Разделы для обновления:** §2.4, §3.2 «Реализация», §4 «Перспективы развития».

---

## 2. Ручная загрузка файла вместо IMAP Gmail

**Изменение:** вместо автоматического приёма писем с вложениями через IMAP реализована ручная загрузка файла решения в разделе «Проверка». Кнопка «Импорт из почты» в настройках — демонстрационная заглушка.

**Причина:** исключение хранения паролей и токенов, упрощение демонстрации на защите, отказ от устаревшего JavaMail и нестабильных схем авторизации Gmail.

**Что изменить в дипломе:** в §2.6 «Интеграция с внешними сервисами» описать модуль импорта из почты как расширяемый компонент; в демонстрационной версии — локальная загрузка файла преподавателем.

**Разделы для обновления:** §1.2 «Анализ аналогов», §2.6, §3.3 «Тестирование».

---

## 3. Заглушка экспорта в Google Sheets

**Изменение:** вместо реальной выгрузки в Google Sheets реализован экспорт результатов в CSV и демонстрационная кнопка «Экспорт в Google Sheets».

**Причина:** стабильность демонстрации, отсутствие необходимости в OAuth и API-ключах на защите.

**Что изменить в дипломе:** указать, что основной формат выгрузки — CSV; интеграция с Google Sheets спроектирована как подключаемый модуль.

**Разделы для обновления:** §2.6, §4 «Перспективы развития».

---

## 4. Упрощённый процесс проверки

**Изменение:** проверка выполняется для одного выбранного файла и одного задания за раз, без очередей писем, пакетной обработки и повторного прогона по email.

**Причина:** наглядность для диплома, простота архитектуры, удобство скриншотов и пошаговой демонстрации.

**Что изменить в дипломе:** в §2.5 «Модуль проверки» описать пошаговый сценарий: выбор задания → ввод данных студента → выбор файла → запуск проверки.

**Разделы для обновления:** §2.5, §3.2.

---

## 5. Отказ от проверки на плагиат

**Изменение:** модуль попарного сравнения исходного кода (Levenshtein) не реализован.

**Причина:** сокращение объёма MVP для ВКР; фокус на основной функции — автоматической проверке по тестам.

**Что изменить в дипломе:** в §1.2 при сравнении с TAPChecker отметить, что антиплагиат вынесен в перспективу; в §4 описать как направление развития.

**Разделы для обновления:** §1.2, §4.

---

## 6. Демонстрационный режим

**Изменение:** добавлен явно видимый демонстрационный режим, симулирующий проверку без реальной компиляции.

**Причина:** гарантированная демонстрация на защите при отсутствии GHC или проблем с PATH.

**Что изменить в дипломе:** в §3.3 «Тестирование и демонстрация» описать демо-режим как инструмент презентации; указать, что в рабочем режиме используется реальная проверка через `javac`/`java` и `runghc`.

**Разделы для обновления:** §3.3, приложение «Руководство пользователя».

---

## 7. Maven и Java 17 вместо IntelliJ-сборки

**Изменение:** проект собирается через Maven (`mvn clean javafx:run`), используется Java 17.

**Причина:** воспроизводимая сборка, соответствие современным требованиям, удобство запуска на любом компьютере.

**Что изменить в дипломе:** в §2.1 «Средства разработки» указать JDK 17, Maven, JavaFX Maven Plugin.

**Разделы для обновления:** §2.1, приложение «Инструкция по запуску».

---

## 8. Изолированный запуск через ProcessBuilder

**Изменение:** вместо долгоживущей сессии `ghci` и `JavaCompiler` API используется запуск отдельных процессов `javac`, `java`, `runghc`/`ghc` с таймаутом 5 секунд и белым списком команд.

**Причина:** проще отлаживать, меньше утечек состояния между проверками, понятнее для описания в дипломе.

**Что изменить в дипломе:** в §2.5 описать схему: компиляция → запуск → сравнение вывода → формирование статуса.

**Разделы для обновления:** §2.5, §2.3 «Диаграмма компонентов».

---

## 9. JSON и CSV вместо Java Serialization

**Изменение:** настройки и задания — JSON (Gson), результаты — CSV.

**Причина:** человекочитаемость, простота резервного копирования и отладки.

**Что изменить в дипломе:** в §2.4 указать форматы хранения и структуру файлов в каталоге `data/`.

**Разделы для обновления:** §2.4, приложение «Структура данных».

---

## 10. Русскоязычный интерфейс и оформление для ВКР

**Изменение:** весь UI на русском языке; спокойная цветовая схема; раздел «О программе» с данными автора.

**Причина:** требования ВКР и удобство скриншотов для пояснительной записки.

**Что изменить в дипломе:** вставить скриншоты разделов в §3.2 и приложение «Руководство пользователя».

**Разделы для обновления:** §3.2, приложения.
